# DSA Practice

Daily Data Structures & Algorithms practice following the [NeetCode 150](https://neetcode.io/practice) problem set. Goal: build consistent pattern recognition, not just grind volume. 1–2 problems/day, ~1 hour, committed every day.

## My Method (every problem, no skipping)

1. **Restate** the problem in my own words (30s). Can't restate it → reread it.
2. **Solve by hand** on a tiny example (3 elements, not 10) — write every step. This is where the algorithm comes from.
3. **Pattern ID** — which bucket? Two Pointers · Sliding Window · Hashmap · BFS/DFS · DP · Greedy · Binary Search.
4. **Pseudocode** in plain words before any real syntax.
5. **Time cap** — 20–25 min stuck → read the editorial/hint, understand fully.
6. **Re-implement next day** from scratch (reading ≠ knowing).
